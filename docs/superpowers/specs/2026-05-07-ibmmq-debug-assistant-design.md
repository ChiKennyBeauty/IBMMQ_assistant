# IBMMQ 调试助手设计文档（软著极速版 MVP）

## 1. 目标与定位

本项目目标是尽快交付一个可演示、可截图、可稳定运行的 `IBMMQ` 桌面调试助手，用于软著材料与首版落地。  
技术栈固定为 `Qt Widgets + Qt 5.14.2 + C++`，平台仅 `Windows`。

核心定位：**方便、简单、能解决日常调试闭环**。

## 2. MVP 范围（只保留必要功能）

第一版仅保留 4 个能力：

1. **连接管理**
   - 支持本机 `bindings` 与远程 `client` 连接。
   - 支持多连接配置保存与切换（开发/测试/生产）。
   - 先支持基础认证（用户名/密码可选）。

2. **对象浏览**
   - 仅展示 `Queue` 与 `Channel` 两类对象。
   - 支持列表刷新与基本详情查看。

3. **消息调试**
   - 队列消息浏览（分页仅 `Next`）。
   - 消息内容文本/HEX 查看。
   - 当前页关键词过滤搜索。

4. **诊断能力**
   - 一键连接测试。
   - 通道状态查看（只读）。

## 3. 明确移出（后续版本再做）

为保证“越快出一版越好”，以下内容从首版移出：

- Topic/Subscription 管理。
- 通道创建/修改/删除（PCF 写操作）。
- 脚本引擎（QJSEngine 与 `mq.*` API）。
- 双向 TLS/mTLS（证书链深度校验）。
- RFH2 深度解析与复杂结构化搜索。
- 跨页/全量消息深度搜索。

## 4. 最小架构

采用两层半结构，降低复杂度：

1. **UI 层（Qt Widgets）**
   - 左侧：连接与对象树（Queue/Channel）。
   - 右侧：对象详情与消息面板。
   - 底部：日志输出。

2. **后台 Worker（单线程执行器）**
   - 负责所有 MQ 调用，避免 UI 卡顿。
   - 统一处理连接、对象读取、消息浏览、状态查询。

3. **工具函数（轻量）**
   - 文本/HEX 转换。
   - 错误码转可读文案。

> 首版不引入脚本模块、不引入多套管理命令框架。

## 5. MQ 接口策略

- 以 `IBM MQ C++ classes` 为主（保持实现简单）。
- 聚焦队列与通道的读操作和消息浏览能力。
- 管理写操作暂不纳入 MVP。

## 6. 交互与数据规则

### 6.1 分页与搜索

- 分页：仅 `Next`（顺序向后浏览）。
- 搜索：仅当前页过滤（不跨页扫描）。

### 6.2 错误展示

统一返回结构：

- `code`
- `reasonText`
- `where`

并在日志区展示“失败原因 + 建议动作”简述。

## 7. 验收标准（软著版）

满足以下即视为首版完成：

1. 能保存连接配置并成功连接本机/远程 MQ。
2. 能看到 Queue 与 Channel 列表及基础详情。
3. 能浏览队列消息（Next 分页、文本/HEX、当前页搜索）。
4. 能执行连接测试并输出明确结果。
5. 整体交互无明显卡死，异常不崩溃。

## 8. 开发顺序（最快落地）

1. Qt 工程骨架 + 连接配置页。
2. MQ 连接与连接测试。
3. Queue/Channel 列表读取。
4. 消息浏览面板（Next 分页 + 当前页搜索 + 文本/HEX）。
5. 错误文案、日志、稳定性收口。

## 9. 软著材料准备建议

围绕 MVP 输出材料：

- 功能说明：连接、对象浏览、消息调试、诊断四大模块。
- 界面截图：连接页、对象树、消息浏览页、测试结果页。
- 技术说明：Qt/C++ 桌面架构、后台 Worker 防阻塞设计。

## 10. 结论

本设计以“最小可用、最快交付”为第一目标，确保在短周期内交付一个真正可用的 IBMMQ 调试助手首版，并为后续扩展（脚本、PCF 写操作、Topic/Subscription、TLS 增强）保留演进空间。
