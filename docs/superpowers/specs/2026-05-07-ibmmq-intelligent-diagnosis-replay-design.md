# IBMMQ 智能诊断与会话复盘助手设计文档（软著创新版）

## 1. 目标与定位

在现有 `IBMMQ 调试助手` 的基础上，新增两类创新能力：

- A: 故障指纹诊断引擎
- C: 调试会话回放与复盘

目标是形成“可落地、可演示、可申报”的软著增强版本，在 1-2 周内完成首版交付。

## 2. 创新主线（A + C）

### 2.1 主创新 A：故障指纹诊断引擎

- 输入：连接参数、MQ 返回码、错误文本、操作上下文
- 输出：故障类型、置信度、证据链、修复建议
- 价值：从“仅报错”升级为“可解释诊断”

### 2.2 辅创新 C：会话回放与复盘

- 自动记录调试动作序列和结果
- 支持关键只读动作重放
- 输出结构化复盘报告
- 价值：将一次排障过程沉淀为可复用资产

### 2.3 组合优势

- A 负责“判断问题是什么”
- C 负责“保留证据和复现过程”
- 形成闭环：发现 -> 解释 -> 处置 -> 复盘

## 3. 架构设计与数据流

在当前 `MainWindow <-> MqWorker` 主链路不变前提下，新增 4 个模块。

### 3.1 DiagnosisRuleEngine

- 负责规则加载、条件匹配、评分、建议生成
- 输入：`MqError`、操作类型、配置快照
- 输出：`DiagnosisResult`

### 3.2 SessionRecorder

- 负责会话生命周期与事件记录
- 记录对象：连接测试、对象刷新、消息浏览、发送、收取、诊断

### 3.3 ReplayService

- 读取会话事件流并重放关键动作
- 首版仅支持只读重放：连接测试、对象刷新、消息浏览

### 3.4 ReportBuilder

- 生成复盘报告（首版 Markdown）
- 汇总诊断结论、证据链与建议动作

### 3.5 数据流

1. UI 发起 MQ 操作  
2. `MqWorker` 执行并返回结果  
3. `SessionRecorder` 记录事件  
4. 失败或异常时进入 `DiagnosisRuleEngine`  
5. UI 展示诊断结论与建议  
6. `ReportBuilder` 导出会话复盘报告  
7. `ReplayService` 可按历史会话进行重放

## 4. 规则模型与评分机制（技术核心）

### 4.1 规则结构

每条规则定义如下字段：

- `ruleId`: 规则编号（如 `MQ-CONN-2009`）
- `scene`: 场景（connect/list/browse/send/receive）
- `conditions`: 触发条件集合
- `weights`: 条件权重
- `diagnosis`: 故障结论
- `actions`: 建议动作列表
- `severity`: 严重级别（P1/P2/P3）

### 4.2 匹配维度

- 错误码维度：MQ reason code
- 文本维度：错误关键词匹配
- 配置维度：连接地址/通道/队列参数合法性
- 行为维度：短时间重复失败模式

### 4.3 评分与置信度

- 评分公式：`Score = Σ(命中条件 * 条件权重)`，归一化 0~100
- 置信度分级：
  - 高：`>= 80`
  - 中：`50~79`
  - 低：`< 50`

低置信度场景必须输出“下一步检查项”，避免误导性结论。

### 4.4 结果结构

- `faultType`: 故障类型
- `confidence`: 置信度
- `evidence`: 命中证据列表
- `recommendedActions`: 建议动作（按优先级）
- `nextCheck`: 下一步排查建议（低置信时必填）

## 5. 会话回放与复盘模型

### 5.1 会话模型（Session）

- `sessionId`
- `startAt/endAt`
- `envTag`（dev/test/prod）
- `events[]`
- `summary`

### 5.2 事件模型（Event）

- `eventType`
- `timestamp`
- `inputSnapshot`（脱敏）
- `result`（ok/code/reason）
- `durationMs`
- `diagnosisRef`（可空）

### 5.3 重放机制（首版）

- 模式：
  - 单步重放（step-by-step）
  - 快速重放（fast）
- 输出：本次执行结果与历史结果差异

### 5.4 复盘报告结构

- 会话基本信息
- 故障摘要
- 关键事件时间线
- 智能诊断与证据
- 建议动作与执行结果
- 重放差异与结论

## 6. UI 扩展方案

在现有界面基础上增加两个区域：

- 智能诊断面板：展示故障类型、置信度、建议动作
- 会话复盘面板：历史会话列表、回放触发、报告导出

不调整现有核心交互，只做增量扩展，降低回归风险。

## 9. 验收标准（创新版）

满足以下即视为 A+C 首版完成：

1. 常见故障可输出诊断类型与置信度
2. 诊断结果包含证据与建议动作
3. 会话事件可完整记录并查询
4. 至少 3 类只读动作支持重放
5. 可导出完整复盘报告
6. 现有功能不回退

## 10. 风险与边界

- 本版不引入复杂机器学习模型，采用可解释规则机制
- 回放仅支持只读动作，不包含管理写操作
- 规则库先覆盖高频问题，后续迭代扩展

该边界确保交付可控、可测试、可用于软著首版申报。
