# BMMQ通讯调试助手软件 源码说明文档

## 1. 文档说明

- 软件名称：BMMQ通讯调试助手软件
- 版本号：V1.0
- 说明目的：用于软件著作权申请时说明程序结构、核心模块及主要实现逻辑

## 2. 技术架构

本软件采用 C++17 + Qt5 Widgets 开发，界面层与 MQ 通讯处理通过信号/槽与工作线程解耦，整体为桌面端单机应用架构。

- 界面层：`MainWindow`
- MQ 工作层：`MqWorker`
- 诊断层：`DiagnosisRuleEngine`
- 会话层：`SessionRecorder`、`ReplayService`
- 报告层：`ReportBuilder`

## 4. 主要模块说明

### 4.1 主界面模块（MainWindow）

- 负责界面布局、用户输入、按钮事件处理、日志显示与诊断显示。
- 通过 Qt 信号向 `MqWorker` 发起连接、对象刷新、消息浏览、发送、接收请求。
- 对操作结果进行处理并记录会话事件。

主要功能函数：

- `onConnected()`：处理连接结果
- `onObjectsListed()`：处理对象列表结果
- `onMessagesBrowsed()`：处理分页浏览结果
- `onMessageSent()`：处理发送结果
- `onMessageReceived()`：处理接收结果
- `runReplay()`：执行会话回放
- `validateConnectionInputs()`：连接参数合法性校验

### 4.2 MQ 工作模块（MqWorker）

- 封装 IBM MQ 通讯调用，避免阻塞 UI。
- 基于 IBM MQ C++ 接口执行连接、队列打开、消息收发与浏览。

主要接口：

- `connectToQueueManager()`：连接测试
- `listObjects()`：对象列表获取
- `browseMessages()`：分页浏览消息
- `sendMessage()`：发送消息
- `receiveMessage()`：接收消息

### 4.3 诊断规则模块（DiagnosisRuleEngine）

- 根据场景、错误码、错误文本生成诊断结果。
- 输出内容包括：故障类型、置信度、证据链、建议动作。

当前实现示例：

- 针对 `connect` 场景 + 错误码 `2009` + 超时关键词，识别为连接超时问题并给出网络与超时配置建议。

### 4.4 会话记录与回放模块

- `SessionRecorder`：记录每次操作的输入快照、结果状态、错误信息与时间戳。
- `ReplayService`：只读回放关键事件（connect/list/browse）以支持调试复盘。

### 4.5 报告生成模块（ReportBuilder）

- 将会话数据和诊断结果组装为 Markdown 报告文本。
- 由主界面触发导出并写入 `reports` 目录。

## 5. 核心数据结构

### 5.1 MQ 结果结构（MqTypes）

- `ConnectResult`：连接是否成功及错误信息
- `ObjectListResult`：对象列表及错误信息
- `BrowseResult`：消息分页结果、是否有更多数据、下一个偏移量
- `SendResult`：发送目标队列、字节数及错误信息
- `ReceiveResult`：是否有消息、消息内容及错误信息

### 5.2 诊断结构（DiagnosisTypes）

- `DiagnosisContext`：诊断输入上下文（场景、错误码、错误文本、扩展属性）
- `DiagnosisResult`：诊断输出（匹配状态、故障类型、置信度、证据、建议）

### 5.3 会话结构（SessionTypes）

- `SessionEvent`：单步操作事件
- `SessionData`：会话总体数据
- `ReplayDelta`：回放差异项

## 6. 程序流程说明

1. 程序启动，注册元类型并初始化主界面。
2. 主界面创建 `MqWorker` 并迁移至独立线程。
3. 用户发起操作，界面层发送请求信号到 `MqWorker`。
4. `MqWorker` 调用 MQ 接口执行操作并返回结果信号。
5. 主界面更新显示、记录会话、失败时触发诊断。
6. 用户可进行会话回放或导出调试报告。

## 7. 构建与测试说明

```bash

cmake --build build --config Debug
```

```bash

```

测试覆盖连接配置、消息格式化、元类型注册、诊断规则、会话记录、报告构建等基础能力。
