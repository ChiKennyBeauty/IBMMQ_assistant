# BMMQ通讯调试助手软件 用户手册

## 1. 文档说明

- 软件名称：BMMQ通讯调试助手软件
- 版本号：V1.0
- 文档用途：指导用户完成安装、连接配置、消息调试、故障诊断与报告导出
- 适用对象：IBM MQ 运维人员、开发测试人员、实施工程师

## 2. 软件概述

BMMQ通讯调试助手软件是一款基于 Qt 的桌面调试工具，面向 IBM MQ 通讯联调场景，提供连接测试、对象查看、消息浏览、消息发送、消息接收、会话回放、诊断建议和报告导出功能，帮助用户快速定位 MQ 通讯问题并沉淀调试记录。

## 3. 运行环境

### 3.1 硬件环境

- CPU：双核及以上
- 内存：4GB 及以上，推荐 8GB
- 磁盘：剩余空间 500MB 及以上

### 3.2 软件环境

- 操作系统：Windows 10/11（64 位）
- 运行库：Qt5 Widgets 相关运行环境
- IBM MQ 客户端/SDK：可用的 IBM MQ C/C++ 运行库

## 4. 安装与启动

### 4.1 安装步骤

1. 准备应用程序可执行文件及依赖库。
2. 确认 IBM MQ 客户端库可被程序正确加载。
3. 将程序目录放置在本地磁盘任意路径。

### 4.2 启动步骤

1. 双击应用程序主程序。
2. 启动后进入主界面，显示连接参数区、对象区、消息区、日志区和诊断区。

## 5. 功能说明

### 5.1 连接测试

- 功能入口：`测试连接`按钮
- 输入参数：
  - 队列管理器（Queue Manager）
  - 通道（Channel）
  - 连接地址（ConnName，格式：`host(port)`）
- 操作说明：
  1. 填写参数后点击`测试连接`。
  2. 成功时日志显示“连接成功”。
  3. 失败时日志显示错误信息，并自动触发诊断建议。

### 5.2 对象刷新

- 功能入口：`刷新对象`按钮
- 功能描述：获取并显示队列/通道等对象列表，用于后续调试操作确认目标对象。

### 5.3 消息浏览与分页

- 功能入口：`浏览消息`、`下一页`按钮
- 功能描述：
  - 支持按页浏览指定队列消息
  - 每页展示消息预览文本
  - 支持当前页关键字过滤
- 操作说明：
  1. 在“浏览队列”输入队列名。
  2. 点击`浏览消息`查看第一页。
  3. 若有后续数据，点击`下一页`翻页。

### 5.4 消息发送

- 功能入口：发送区域 + `发送消息`按钮
- 功能描述：向指定队列发送 UTF-8 文本消息。
- 操作说明：
  1. 填写“发送队列”。
  2. 输入发送内容。
  3. 点击`发送消息`。
  4. 日志显示发送结果和字节数。

### 5.5 消息接收

- 功能入口：`收取一条`按钮
- 功能描述：从指定队列接收一条消息并展示预览。
- 操作说明：
  1. 确认浏览队列名。
  2. 点击`收取一条`。
  3. 如有消息则展示；无消息时提示“暂无消息”。

### 5.6 故障诊断

- 功能描述：在连接、浏览、发送、接收失败时，自动输出故障类型、置信度和建议动作。
- 诊断输出位置：诊断文本框
- 示例场景：连接超时、网络不稳定等问题提示。

### 5.7 会话回放

- 功能入口：`回放会话`按钮
- 功能描述：对当前会话的关键事件执行只读回放，便于复盘调试流程。

### 5.8 报告导出

- 功能入口：`导出报告`按钮
- 功能描述：导出会话与诊断信息为 Markdown 报告文件。
- 输出目录：`reports/`
- 文件命名：`replay-report-YYYYMMDD-HHMMSS.md`

### 5.9 主题切换

- 功能入口：`切换到暗色/浅色`按钮
- 功能描述：支持浅色与暗色主题切换，提升不同光照条件下的可读性。

## 6. 操作流程（推荐）

1. 填写连接参数并执行`测试连接`。
2. 执行`刷新对象`确认目标队列/通道。
3. 通过`浏览消息`和`下一页`查看历史消息。
4. 使用发送区发送测试消息。
5. 点击`收取一条`验证消费结果。
6. 观察日志与诊断建议，必要时执行`回放会话`。
7. 调试完成后点击`导出报告`留档。

## 7. 参数校验规则

- 队列管理器不能为空。
- 通道不能为空。
- 连接地址格式必须为 `host(port)`，例如 `127.0.0.1(1414)`。
- 浏览队列、发送队列和发送内容在对应操作中不能为空。

## 8. 常见问题处理

### 8.1 连接失败

- 检查队列管理器、通道、连接地址是否填写正确。
- 检查端口连通性和防火墙策略。
- 检查 IBM MQ 服务是否正常启动。

### 8.2 浏览失败

- 检查队列名称是否存在。
- 检查当前账号是否有浏览权限。

### 8.3 发送失败

- 检查目标队列权限及队列状态。
- 检查消息体是否为空。

### 8.4 接收无消息

- 确认队列当前是否有可消费消息。
- 确认生产端是否已写入消息。

## 9. 维护与支持

- 建议定期保存导出报告，便于问题追踪与知识沉淀。
- 建议在测试环境先验证连接参数，再应用到生产联调环境。
